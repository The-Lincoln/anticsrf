What is SecureUploader?

A PHP class that turns insecure file uploads into a reliable, secure, and developer‑friendly process.

Built with Object Oriented Programming (OOP) principles, SecureUploader validates files by their real MIME type (not just extension), enforces size limits, optionally checks image dimensions, and returns a clean UploadResult object. It follows all scit.zya.me contribution guidelines – includes at least one class, provides examples, and respects user tracking options.

Zero external dependencies
Works with any PHP 7.4+ project
Fully tested & documented
$uploader = new SecureUploader();
$uploader->allowMimeTypes(['image/jpeg', 'application/pdf'])
    ->setMaxSize(5 * 1024 * 1024)
    ->setMaxImageDimensions(1920, 1080);

$result = $uploader->upload($_FILES['document']);
if ($result->success) {
    echo "Uploaded to: " . $result->filePath;
} else {
    echo "Error: " . $result->errorMessage;
}

🔐 Key Features & Benefits

Real MIME Validation

Uses PHP's finfo to detect the actual file type – not the browser‑supplied $_FILES['type']. Blocks spoofed extensions (e.g., .jpg.exe).

Size & Dimension Limits

Enforce maximum file size (in bytes) and optional image width/height constraints – perfect for avatars, banners, or gallery uploads.

Atomic Write + Sanitization

Prevents file overwriting by appending a counter; sanitizes filenames to remove dangerous characters; uses temp + rename for safety.

Clean Result Object

Returns UploadResult with success, filePath, and errorMessage. No exceptions – easy to integrate.

Git / SVN / CVS Ready

Native support for daily imports from version control. The SCIT platform automatically pulls updates and notifies users.

Email Notifications

Once approved on scit.zya.me, tens of thousands of developers receive an email – guaranteed audience for your work.

📘 Step‑by‑Step Implementation Guide

1
Download the package from SCIT
Visit scit.zya.me, search for “SecureUploader”, and download the .zip or .tar.gz archive. (Registration may be required depending on the author’s tracking option – but examples/docs are always public.)
2
Include the class in your project
Copy classes/SecureUploader.php into your project (e.g., lib/). Then require it:
require_once 'lib/SecureUploader.php';
3
Instantiate & configure
Set allowed MIME types, max size, and upload directory:
$uploader = new SecureUploader();
$uploader->allowMimeTypes(['image/jpeg', 'image/png', 'application/pdf'])
    ->setMaxSize(10 * 1024 * 1024)
    ->setUploadDir('uploads/avatars/')
    ->setMaxImageDimensions(800, 600);
4
Process the uploaded file
In your form handler, call upload() and check the result:
if ($_SERVER['REQUEST_METHOD'] === 'POST' && isset($_FILES['myfile'])) {
    $result = $uploader->upload($_FILES['myfile']);
    if ($result->success) {
        echo "File saved to " . htmlspecialchars($result->filePath);
    } else {
        echo "Upload failed: " . $result->errorMessage;
    }
}
5
Test & deploy
Run a test upload – the class will create the upload directory automatically. For production, ensure the directory has write permissions (e.g., chmod 755). That’s it!
Pro Tips
  • Always store uploaded files outside the webroot if possible, or use a .htaccess to block script execution.
  • Combine SecureUploader with a CSRF token for maximum security.
  • Use setMaxImageDimensions() only for image types – it’s automatically ignored for PDFs.
  • The sanitizeFilename() method (default true) replaces unsafe characters with underscores.
